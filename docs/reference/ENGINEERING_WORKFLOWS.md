# Engineering Workflows

Lifeline's governance workflows sit above skills, agents, and teams and define repeatable execution paths for governance review.

## Phase order

1. skills
2. agents
3. teams
4. workflows

## Current governance workflows

- [documentation-governance-workflow](../../.github/workflows-governance/documentation-governance-workflow.md)
- [cicd-governance-workflow](../../.github/workflows-governance/cicd-governance-workflow.md)

## Workflow intent

- workflows coordinate repeatable governance sequences
- they build on skills, agents, and teams
- they are not the same as GitHub Actions deployment workflows
- they should help Phase 6C and future expansion stay consistent rather than ad hoc
