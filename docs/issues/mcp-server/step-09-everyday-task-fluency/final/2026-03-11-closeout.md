## Key capabilities delivered

1. **Subtask CRUD with stable identity** ‚Äî Each subtask carries a UUID `subtaskId` for reliable agent references. Full add/complete/uncomplete/update/remove operations.
2. **Archive-first task lifecycle** ‚Äî `archive_task` replaces `delete_task` as the primary removal action. Mutation guards prevent accidental edits on archived tasks. `restore_task` and `batch_restore` for recovery.
3. **Window-based planning queries** ‚Äî `list_tasks` (with `window` parameter) supports `this_week`, `next_week`, `this_month`, `next_month`, `overdue`, and `YYYY-MM` tokens. Understands dateRange recurrence spans.
4. **Fuzzy title similarity** ‚Äî `find_similar_tasks` uses pg_trgm trigram matching to detect near-duplicates before creation.
5. **Enterprise hardening** ‚Äî Optimistic concurrency (If-Match/expectedUpdatedAt), correlation IDs (X-Correlation-ID), rate limiting (120 req/min/principal), timestamps in all payloads.
6. **Agent intelligence** ‚Äî Rewritten server instructions with structured guidance, improved tool descriptions.
